# Vault - API Key Manager

本地运行的 API Key 安全管理器，纯浏览器端加密存储，数据不离开本机。

## 快速开始

直接打开 `index.html` 即可使用，无需安装任何依赖。

> 需要支持 Web Crypto API 的现代浏览器（Chrome / Edge / Firefox / Safari），通过 `file://` 或 `https://` 访问。

## 功能概览

### 安全系统

| 特性     | 说明                                    |
| -------- | --------------------------------------- |
| 加密算法 | AES-256-GCM                             |
| 密钥派生 | PBKDF2（SHA-256，100,000 次迭代）       |
| 主密码   | 用于派生加密密钥，加密所有 API Key      |
| 解锁方式 | 手势密码（九宫格）或 文字密码（二选一） |

### 解锁方式

首次设置时可选择：

- **手势密码** — 3×3 九宫格，pointer 事件绘制路径，至少连接 4 个点
- **文字密码** — 使用主密码直接解锁

### API Key 管理

- 添加 / 编辑 / 删除
- 实时搜索（名称、URL、备注）
- 显示 / 隐藏 Key 值
- 一键复制到剪贴板

### 导出

- **导出 Excel** — 将所有 API Key 导出为 `.xlsx` 文件，包含名称、Key、URL、备注、创建时间

### 其他

- 手势密码重置（需验证主密码）
- 全部数据重置（双重确认，清除所有设置和数据）
- 深色主题，卡片布局
- 响应式设计，支持移动端

## 数据存储

使用 `localStorage` 存储，数据仅在当前浏览器中：

| 键名           | 内容                                 |
| -------------- | ------------------------------------ |
| `vault_config` | 加密配置（盐值、解锁方式、验证数据） |
| `vault_data`   | 加密后的 API Key 列表                |

- 自动保存，刷新不丢数据
- 清除浏览器数据会导致所有内容丢失

## 安全说明

- 所有 API Key 使用 AES-256-GCM 加密后存储
- 加密密钥由主密码通过 PBKDF2 派生，不存储明文密码
- 手势密码仅存储 SHA-256 哈希，不可逆推
- 主密钥使用手势派生密钥包裹存储
- 无后端、无网络请求、无数据上传
